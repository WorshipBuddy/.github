# Contributing to WorshipBuddy

Thank you for your interest in contributing to WorshipBuddy! We're a 501(c)(3) nonprofit building free tools for churches, and we welcome help from the community.

## How to Contribute

### Reporting Bugs

- Check existing issues before opening a new one
- Include steps to reproduce, expected vs. actual behavior, and your OS/app version
- Screenshots and logs are always helpful

### Suggesting Features

- Open an issue with a clear description of the feature and the problem it solves
- Explain how it benefits worship teams or churches

### Code Contributions

1. Fork the repository
2. Create a feature branch from the default branch
3. Make your changes with clear, descriptive commits
4. Test your changes locally
5. Open a pull request with a summary of what you changed and why

### Documentation

Improvements to documentation, guides, and README files are always appreciated.

## Development Setup

Each repository has its own setup instructions in its README. Generally:

- **PresenterBuddy** — Node.js + Electron (React/TypeScript)
- **PresenterBuddy Studio** — Node.js + Electron (Svelte/TypeScript)
- **ChurchBuddy App** — React Native
- **WorshipBuddy API** — Python (Django/FastAPI)

## Guidelines

- Follow the existing code style in each project
- Keep pull requests focused — one feature or fix per PR
- Write meaningful commit messages
- Be respectful and constructive in all discussions

## Questions?

Reach out to us at **support@worshipbuddy.org** or open a discussion on the relevant repository.
